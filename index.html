The Problem

Current Limitations

  • • Files stored on IPFS/Walrus are either publicly visible to everyone or completely private
  • • No native way to monetize encrypted content
  • • Creators must trust centralized platforms for paid access control

Example Scenario

When Alice wants to monetize her video content:

  • • She can't upload unencrypted - anyone could access without paying
  • • If she uploads encrypted, buyers must trust she'll provide the key
  • • No trustless way to handle pay-for-access

Enter Filena's Solution

Proxy Re-Encryption (PRE)

A revolutionary cryptographic method that enables:

  • Trustless file sharing with built-in encryption
  • Automated access control via smart contracts
  • Pay-to-access without intermediaries

How It Works

  1. Creator encrypts file with their public key
  2. Buyer makes payment through Solana smart contract
  3. Contract generates re-encryption key for buyer
  4. Buyer accesses content with their own private key
